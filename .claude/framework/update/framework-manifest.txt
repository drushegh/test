# Framework Manifest
#
# Paths listed here are framework-owned — they're overwritten on every
# update. Paths NOT listed are project-owned and never touched.
#
# Two kinds of entries:
#
#   Directory entries (trailing slash) — "dir mirror"
#     The entire contents of the directory are replaced with upstream's
#     version. Any files you add to these dirs will be destroyed on
#     update. Use this for directories that are 100% framework-owned,
#     where the SUBDIR NAME itself is the ownership boundary
#     (e.g., .claude/agents/framework/, .claude/framework/doctor/).
#
#   File entries (no trailing slash) — "file overwrite"
#     Only that specific file is replaced with upstream's version.
#     Sibling files in the same directory are untouched. Use this
#     for SHARED directories where framework and project content
#     coexist (e.g., .claude/commands/, .claude/hooks/).
#
# If you add new framework-shipped files to a shared directory, list
# them individually here so they propagate to downstream projects.
# New project files in shared dirs are automatically safe (not listed →
# not touched).
